DevOps Intern

Acies Global, Chennai - container scaling and cloud ETL, measured in other people’s time.

What I did

  • Spearheaded a software scaling project using Docker and Kubernetes, optimizing resource allocation and container workflows to improve application performance by 25% and reduce manual configuration time by 30%.
  • Designed and executed end-to-end ETL pipelines in PySpark, integrated with AWS Glue, enabling extraction, transformation and loading from multiple sources into cloud environments.

Context

My first industry role, in Chennai, and my first exposure to software that other people depend on being up. Everything I had written before that summer had exactly one user.

Both workstreams were about the same thing from opposite ends: making the boring parts repeatable. Container orchestration removes the manual step in getting a service running; an ETL pipeline removes the manual step in getting data somewhere useful. Neither produces anything a user sees.

What I took away

Infrastructure work is measured in someone else’s time. The number I am proudest of from that summer is the 30% reduction in manual configuration, not the 25% performance gain - because that one is time returned to engineers, every week, indefinitely. It is the first work I did where the value compounded rather than being delivered once.

Containers are where "works on my machine" stops being a joke and starts being a specification. Writing a Dockerfile forces you to state every assumption your environment was quietly making on your behalf, and most of the debugging is discovering assumptions you did not know you had.
