Automated Artist

A two-axis plotter that draws: STM32H7 firmware turning a serial vector stream into pen motion.

Summary

  • Wrote firmware on an STM32H7 that parses and executes drawing commands arriving over a serial (UART) interface, defining a communication protocol for the link rather than inheriting one.
  • Programmed a 2-axis robotic plotter with custom driver code for NEMA 17 stepper motors, translating incoming serial data into fluid, visually accurate vector drawings on a 2D plane.

The problem

A drawing is continuous and a stepper motor is not. Every curve the machine produces is a lie told convincingly enough in discrete increments, and the whole project lives in how well that lie holds up at the scale of a pen tip.

The second problem is division of labour. Deciding where the pen should go is a vision and geometry problem that wants floating point and a real CPU. Getting the pen there on time is a hard real-time problem that wants a microcontroller and nothing else running. Those are different machines, so the interesting design question is what crosses the wire between them.

How it works

The host does the thinking: an image is reduced to contours and those contours become ordered vector paths. The board does the moving. What crosses the UART is a stream of drawing commands in a protocol I defined - framed, parseable one command at a time, and small enough that the firmware never needs to buffer a whole drawing to start one.

On the board, each command becomes coordinated motion on two axes. The stepper driver code is written against the motors directly rather than through a motion library, because the timing relationship between the two axes is the line quality - the axes have to arrive together or the stroke bows.

What I’d change

The motion is open loop. A stepper that misses a step under load does not report it, so the firmware’s idea of where the pen is drifts away from where the pen actually is, and nothing in the system can tell. Encoders and a homing routine would turn a silent failure into a detectable one, which matters more than the accuracy gain.

The serial protocol is also the coupling point. It was designed for this plotter, which means any change to the mechanism is a change to the protocol and therefore a change on both sides of the wire.
