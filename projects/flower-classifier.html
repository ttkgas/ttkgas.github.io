Summary

  • Developed and deployed a custom image-classification model onto a resource-constrained STM32 microcontroller using TensorFlow Lite and STM32CubeAI, reaching 95% accuracy across three flower types.
  • Engineered a hardware-acceleration path by integrating a Google Coral Edge TPU with the STM32 platform, more than doubling system performance from 3 FPS to 7 FPS.

The problem

Getting 95% on three flower classes is not hard. Getting 95% on a part with no operating system, no GPU, and memory measured in kilobytes is a different problem entirely, and it is a deployment problem rather than a modelling one.

The constraint that governs everything is that the model has to fit - in flash, and then in the RAM the part actually has left over after the rest of the firmware. Accuracy you cannot load is zero accuracy, so the design starts from the memory budget and works backwards toward the network.

How it works

Training happens off-device, where floating point and a real GPU are available. The model is then quantized and passed through STM32CubeAI, which emits C that the normal toolchain compiles straight into the firmware image. That is the part worth noticing: on the microcontroller, inference is a function call inside the application, not a service to be talked to. There is no runtime to schedule around and no dependency to install.

The acceleration path adds a Google Coral Edge TPU alongside the STM32, with the microcontroller staying the host - it sequences capture, hands the tensor across, and takes the result back. Throughput went from 3 FPS to 7 FPS, measured end to end rather than as raw kernel time.

What I’d change

The speedup is real and it is not free: it costs a second chip, a link between them, and the power budget of both. Whether an external accelerator belongs in a design whose entire premise was "on device" is a fair question, and the honest answer depends on a duty-cycle number I never measured.

Three classes is also a forgiving benchmark. The interesting behaviour is what the model does when shown a flower that is none of the three, and a classifier with no rejection option will always answer confidently. A calibrated confidence threshold and a fourth "none of these" outcome would say far more about whether this works than another point of accuracy would.
