Chemistry patents describe reactions in prose. This project turns that prose into machine-readable structure — which strings are starting materials, reagents, solvents and products; which action verbs are reaction events; which chemical belongs to which event — using an LLM constrained by a fixed, closed schema.

The real research question isn’t “can it extract.” It’s how much of the remaining error is fixable by writing better schema definitions, versus how much genuinely requires learning from labeled data. Every experiment here exists to answer that with numbers.

The finding that shaped everything

Scored against the official ChEMU metric, exact-match F1 and relaxed F1 differ by exactly one thing: boundary errors. That gap turns out to be diagnostic.

For NER, exact sat at 0.748 while relaxed hit 0.906 — a 0.158 gap, meaning 54% of errors were the model finding the right entity with the wrong edges. Pure formatting. A deterministic boundary-trimming pass recovered +0.063 for zero model cost.

For event extraction, exact and relaxed were identical to three decimals on both models. Not a single boundary point to recover. Every remaining EE error is the model genuinely misreading the chemistry — whether dissolved is a reaction step or a workup depends on surrounding context.

NER fails on formatting. EE fails on understanding. That single distinction is why the fine-tuning effort targets EE and not NER.

What the experiments established

  • Definition refinement saturates. Four rounds of rewriting schema definitions moved NER exact F1 by less than 0.02 after the second round.
  • Examples beat rules. Adding worked few-shot examples gained +0.158 in a single step — more than four rounds of rule-writing combined.
  • The prompt carries the performance, not the model. A frontier model with a deliberately naive prompt scored 0.718, roughly matching a far smaller model with the same naive prompt.
  • Prompts transfer but don’t transplant. Prompts tuned on one model scored measurably lower on another, even a stronger one.
  • Almost all gains came from restraint. Both tasks started precision-poor; improvement came from teaching the model what not to extract, not how to find more.
  • Two-pass EE was abandoned. +0.017 for double the cost and double the latency didn’t justify itself.

Experimental hygiene

Cleanliness of the splits matters more than any single score: train/dev disjoint with all prompt refinement on dev, plus a frozen 200-document test set drawn from documents never inspected during any refinement round, and a further 100-document generalization set disjoint from both.

Because the model outputs text rather than character offsets, offsets are recovered by locating each predicted string in the source, and anything unlocatable is scored as a false positive. Every number here is a slight lower bound, never an inflated one.

Where it stands

Best held-out results: NER 0.910 and EE 0.792, against a supervised leaderboard ceiling of 0.957 / 0.954. The NER gap is 0.047; the EE gap is 0.162 — which is exactly why EE is the fine-tuning target.

In flight: QLoRA fine-tuning for event extraction on 700 training pairs, evaluated on the frozen test set. After that, knowledge distillation using the prompted frontier model as a teacher, and an RLVR loop applying GRPO with the official scorer directly as a verifiable reward — the scorer already returns a scalar F1 per document, so the reward function is effectively already written.
