Wireless Keyboard

An ergonomic keyboard taken from an empty KiCad schematic to a board that types over Bluetooth.

Summary

  • Designed a multi-layer ergonomic keyboard PCB from scratch in KiCad, routing a switch matrix, the nRF52840 microcontroller and USB-C circuitry into a compact custom form factor.
  • Configured and deployed the open-source ZMK firmware to the custom board, bringing up wireless Bluetooth operation and a fully programmable keymap.

The problem

A keyboard is the smallest complete hardware product I could think of. It has a schematic, a layout, a manufacturing step, a power budget, a radio, firmware and a human interface - and unlike most projects at that size, none of those layers can be stubbed out. The board either types or it does not.

It is also the one piece of hardware where the ergonomics are not an abstraction. I use it for hours a day, which makes it the rare project where I am a completely honest user of my own output.

How it works

Electrically it is a diode-isolated switch matrix scanned by the microcontroller, so n keys cost roughly 2√n pins instead of n - the reason a keyboard fits on a small part at all. The nRF52840 was chosen because it is the part ZMK targets: an ARM core and a Bluetooth Low Energy radio in one package, so wireless is not a second board bolted on. USB-C handles power and flashing.

The firmware side is deliberately not custom. ZMK already solves BLE pairing, power management and layer handling properly, so my contribution is the board definition and the keymap - which is where a keyboard actually becomes personal. Writing my own firmware would have been more code and a worse keyboard.

What I’d change

The board and the firmware are coupled through the matrix pin assignment, so a routing convenience taken during layout turns into a devicetree edit later, and the two can silently disagree. Fixing the pin map early and treating it as an interface rather than an outcome of routing would have cost an hour and saved several.

The deeper limitation is the iteration loop. Every ergonomic claim - stagger, angle, thumb-cluster reach - can only be tested after a manufacturing cycle, and by then it is expensive to be wrong. A printed mock-up before ordering the board is the cheap experiment I skipped.
